# ⚙️ SOC Lab Setup Guide

Full reproducible setup documentation for the Home SOC Operations Lab.  
Environment: VirtualBox on Windows 11 host, Kali Linux + Ubuntu VMs on bridged LAN.

---

## Prerequisites

- Oracle VirtualBox 7.x
- Kali Linux ISO (Rolling) — [kali.org/get-kali](https://www.kali.org/get-kali/)
- Ubuntu 22.04 LTS ISO — [ubuntu.com/download](https://ubuntu.com/download/server)
- Minimum host RAM: 16GB recommended (Kali: ~6GB, Ubuntu: 3GB)
- Home LAN with DHCP router (gateway at `192.168.1.1`)

---

## Phase 1 — Kali Linux VM Setup

### 1.1 VirtualBox VM Configuration

```
Name:        Kali-SOC-Manager
OS Type:     Debian (64-bit)
RAM:         6767 MB
Storage:     50 GB (dynamically allocated)
Network:     Bridged Adapter → your physical NIC
```

### 1.2 Static IP Configuration (Kali)

Edit NetworkManager connection to assign `192.168.1.8`:

```bash
# Find connection name
nmcli connection show

# Set static IP
sudo nmcli connection modify "Wired connection 1" \
  ipv4.method manual \
  ipv4.addresses 192.168.1.8/24 \
  ipv4.gateway 192.168.1.1 \
  ipv4.dns "8.8.8.8,8.8.4.4"

# Apply
sudo nmcli connection down "Wired connection 1"
sudo nmcli connection up "Wired connection 1"

# Verify
ip a show eth0
ping -c 3 google.com
```

## Phase 3 — Zeek Installation (Kali)

### 3.1 Install Zeek

```bash
# Add Zeek repo
echo 'deb http://download.opensuse.org/repositories/security:/zeek/xUbuntu_22.04/ /' \
  | sudo tee /etc/apt/sources.list.d/security:zeek.list

curl -fsSL https://download.opensuse.org/repositories/security:zeek/xUbuntu_22.04/Release.key \
  | gpg --dearmor | sudo tee /etc/apt/trusted.gpg.d/security_zeek.gpg > /dev/null

sudo apt update
sudo apt install zeek -y
```

### 3.2 Configure Zeek Interface

```bash
# Edit node config
sudo nano /opt/zeek/etc/node.cfg

# Set interface to eth0:
# [zeek]
# type=standalone
# host=localhost
# interface=eth0

# Deploy and start
sudo /opt/zeek/bin/zeekctl deploy
sudo /opt/zeek/bin/zeekctl start

# Check status
sudo /opt/zeek/bin/zeekctl status

# View live logs
tail -f /opt/zeek/logs/current/conn.log
tail -f /opt/zeek/logs/current/dns.log
```

**Zeek version installed:** 8.0.5

## Phase 4 — Wazuh Agent: Ubuntu VM

### 4.1 Ubuntu VM Configuration

```
Name:        ubuntu-soc-agent
OS:          Ubuntu 22.04 LTS Server
RAM:         2048 MB
Network:     Bridged Adapter
Static IP:   192.168.1.5
```

### 4.2 Set Static IP (Ubuntu)

```bash
sudo nano /etc/netplan/00-installer-config.yaml
```

```yaml
network:
  version: 2
  ethernets:
    enp0s3:
      addresses:
        - 192.168.1.5/24
      routes:
        - to: default
          via: 192.168.1.1
      nameservers:
        addresses: [8.8.8.8, 8.8.4.4]
      dhcp4: false
```

```bash
sudo netplan apply
ip a show enp0s3
```

### 4.3 Install Wazuh Agent (Ubuntu)

```bash
# On Ubuntu — point to Kali manager
curl -s https://packages.wazuh.com/key/GPG-KEY-WAZUH | \
  gpg --no-default-keyring --keyring gnupg-ring:/usr/share/keyrings/wazuh.gpg \
  --import && chmod 644 /usr/share/keyrings/wazuh.gpg

echo "deb [signed-by=/usr/share/keyrings/wazuh.gpg] \
  https://packages.wazuh.com/4.x/apt/ stable main" | \
  sudo tee /etc/apt/sources.list.d/wazuh.list

sudo apt update
sudo WAZUH_MANAGER='192.168.1.8' apt install wazuh-agent -y

# Start agent
sudo systemctl daemon-reload
sudo systemctl enable wazuh-agent
sudo systemctl start wazuh-agent

# Verify on Kali dashboard — agent should appear as Active
```

## Phase 6 — TheHive 5.2 via Docker

### 6.1 Install Docker (Kali)

```bash
sudo apt install docker.io docker-compose -y
sudo systemctl enable docker
sudo systemctl start docker
sudo usermod -aG docker $USER
```

### 6.2 Deploy TheHive

```bash
mkdir ~/thehive && cd ~/thehive

# docker-compose.yml — TheHive + Cassandra + Elasticsearch
cat > docker-compose.yml << 'EOF'
version: "3"
services:
  thehive:
    image: strangebee/thehive:5.2
    ports:
      - "9000:9000"
    depends_on:
      - cassandra
      - elasticsearch
    environment:
      - JVM_OPTS=-Xms512m -Xmx512m

  cassandra:
    image: cassandra:4
    environment:
      - CASSANDRA_CLUSTER_NAME=thehive

  elasticsearch:
    image: docker.elastic.co/elasticsearch/elasticsearch:7.17.9
    environment:
      - discovery.type=single-node
      - xpack.security.enabled=false
      - ES_JAVA_OPTS=-Xms512m -Xmx512m
EOF

# Start TheHive (start on demand, not always-on)
docker-compose up -d

# Access
# URL: http://192.168.1.8:9000
# Default: admin@thehive.local / secret
```

## Troubleshooting

| Problem | Fix |
|---------|-----|
| Kali loses internet after static IP | `sudo nmcli connection down/up "Wired connection 1"` |
| Agent shows disconnected | Restart: `sudo systemctl restart wazuh-agent` |
| Zeek not generating logs | Check interface: `sudo /opt/zeek/bin/zeekctl status` |
| TheHive unreachable | `docker-compose ps` — check if containers are up |
| Wazuh dashboard blank | Wait 2–3 min after starting Elasticsearch |
